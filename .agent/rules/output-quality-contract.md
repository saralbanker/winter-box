# Output Quality Contract

Every output must satisfy its type contract before being delivered:

- **Code output**: working code + explanation of non-obvious decisions + test or usage example
- **Debug output**: root cause identified (not just symptom) + fix applied + validation result
- **Architecture output**: structure description/diagram + decision rationale + trade-offs noted
- **Research output**: conclusion first + supporting evidence + cited sources + confidence level
- **Refactor output**: before/after comparison + tests confirm no regression + what smell was fixed
- No output is "done" until the active skill's Validation Gate checklist is fully checked
